# STYLEE — Setup & Run Guide

A complete Flutter + Firebase e-commerce app for a clothing brand.

---

## STEP 1 — Prerequisites

Make sure you have these installed:

| Tool | Version | Check |
|------|---------|-------|
| Flutter SDK | ≥ 3.19 | `flutter --version` |
| Dart SDK | ≥ 3.3 | `dart --version` |
| Android Studio / Xcode | Latest | For emulator |
| Node.js | ≥ 18 | `node --version` |
| FlutterFire CLI | Latest | `flutterfire --version` |

**Install FlutterFire CLI (if not installed):**
```bash
dart pub global activate flutterfire_cli
```

---

## STEP 2 — Create a Firebase Project

1. Go to https://console.firebase.google.com
2. Click **"Add project"** → name it `stylee-app`
3. Disable Google Analytics (not needed) → **Create project**
4. In the left menu, click **Authentication** → **Get started**
   - Enable **Email/Password** provider → Save
5. Click **Firestore Database** → **Create database**
   - Choose **Start in test mode** (for development)
   - Select a Cloud Firestore location (e.g., `us-central`)
   - Click **Enable**

---

## STEP 3 — Connect Firebase to Flutter

Inside the `stylee/` project folder, run:

```bash
cd stylee
flutter pub get
flutterfire configure
```

When prompted:
- Select the `stylee-app` Firebase project you just created
- Select platforms: **Android** ✓, **iOS** ✓ (skip Web if not needed)

This automatically:
- Creates `lib/firebase_options.dart` (replaces the placeholder)
- Creates `android/app/google-services.json`
- Creates `ios/Runner/GoogleService-Info.plist`

---

## STEP 4 — Set Firestore Security Rules

In Firebase Console → Firestore → **Rules**, paste:

```=
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Products: anyone logged in can read, no one can write from client
    match /products/{id} {
      allow read: if request.auth != null;
      allow write: if false;
    }

    // Users: only the owner can read/write their own data
    match /users/{uid} {
      allow read, write: if request.auth != null && request.auth.uid == uid;

      match /cart/{item} {
        allow read, write: if request.auth != null && request.auth.uid == uid;
      }

      match /orders/{order} {
        allow read, write: if request.auth != null && request.auth.uid == uid;
      }
    }
  }
}
```

Click **Publish**.

---

## STEP 5 — Seed Products into Firestore

Products are seeded automatically when you first open the Home screen.
The `FirestoreService.seedProducts()` method runs once on app launch
and populates the `products` collection — it does nothing if products already exist.

## STEP 6 — Run the App

### Android Emulator
```bash
# Start an Android emulator from Android Studio first, then:
flutter run
```

### iOS Simulator (macOS only)
```bash
flutter run -d iPhone
```

### Physical Device
```bash
flutter devices          # list connected devices
flutter run -d <device-id>
```

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `google-services.json` missing | Run `flutterfire configure` again |
| Products not showing | Check Firestore rules allow reads; wait for seeding |
| `firebase_options.dart` has placeholder values | Run `flutterfire configure` to overwrite |
| Build fails on iOS | Run `cd ios && pod install` |
| `flutter pub get` fails | Check internet connection; run `flutter clean` first |

## How to Create an Admin Account

The Admin Panel is only visible to users with `isAdmin: true` in Firestore.

**Step-by-step:**

1. **Register** a new account in the app (e.g. `admin@stylee.com`)
2. Go to **Firebase Console** → **Firestore Database** → `users` collection
3. Click on the document with your admin user's UID
4. Click **"Add field"** → set field name: `isAdmin`, type: `boolean`, value: `true`
5. Click **Save**
6. **Log out and log back in** to the app — the ADMIN badge and Admin Panel button will appear on the Profile screen

> The admin route `/admin` is protected in `app_router.dart` — any non-admin user attempting to access it is redirected to `/home`.

**What the Admin Panel can do:**
- **Add Product** tab: name, category, gender, price, size quantities (S/M/L/XL/XXL), description, colors with photo URLs, New/Featured flags
- **Products** tab: view all products with delete button per product
- **Reset Demo Data**: deletes all products and re-seeds the original 6 demo products
